00039 /* 
00040     @brief Legacy rig-file importer.
00041 
00042     RoR's physics work on this principle:
00043     1. There is a node array, pre-allocated to some 5000 items.
00044     2. As .truck file is parsed, defined nodes are reserved in this array and accessible through their 
00045        index. Alternatively, nodes can be defined "named"; in that case, a <name->node index> translation 
00046        is performed and node can be addressed by both.
00047     3. Nodes are also generated by some sections (cinecam, wheels...). These can also be adressed by index.
00048 
00049     Traditionally, the order of nodes in nodes-array was determined by order of definitions in .truck file.
00050     This approach had downsides:
00051     1. Nodes could be adressed before they were defined
00052        This could be detected, but for compatibility reasons, parser emitted a warning only.
00053     2. Non-existent nodes could be adressed.
00054        Due to pre-allocated nature of node-array, this passed unnoticed.
00055        It could be explicitly detected, but parsers weren't equipped to do so in every case.
00056     3. Very hard to determine index of some nodes.
00057        Creating rig was traditionally a manual process, and not every section supported named nodes (i.e. axles)
00058        Knowing index of node was necessary, but with named/generated nodes, it was hard to tell.
00059     3. Unfeasible for visual editor with mouse controls.
00060        This approach would, by nature, re-order definitions and make direct index-adressing impossible.
00061 
00062     To ease code maintenance, enable bullet-proof checking and prepare grounds for visual rig editor,
00063     a new approach was applied: Truckfile would be loaded to RigDef::File structure in it's entirety, 
00064     and then parsed in pre-determined order.
00065     Order (section name[number of nodes generated per line]):
00066         1. nodes[1]
00067         2. nodes2[1]
00068         3. cinecam[1]
00069         4. wheels[num_rays*2]
00070         5. wheels2[num_rays*4]
00071         6. meshwheels[num_rays*2]
00072         7. meshwheels2[num_rays*2]
00073         8. flexbodywheels[num_rays*4]
00074     
00075     Naturally, older truckfiles which rely on user-determined order of definitions needed to be converted
00076     with index-references updated to match new node-array positions. This class does exactly that.
00077 
00078 */
